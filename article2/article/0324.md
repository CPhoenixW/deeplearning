## 2. 最新方法详解

### 2.1 FedTrident (2026-03) - 针对标签翻转

**问题**: 标签翻转攻击(Targeted Label-Flipping Attacks, TLFA)在道路状况分类中的威胁

**方法流程**:
```
1. 攻击目标识别
   - 分析恶意客户端的目标(哪些类被翻转)

2. 关键特征提取
   - 从中毒模型中提取对目标类敏感的神经元

3. 神经元级分析
   - 逐神经元比较客户端模型与全局模型
   - 识别异常激活模式

4. GMM聚类
   - 使用高斯混合模型对神经元行为聚类
   - 分离恶意与良性客户端

5. 自适应客户端评分
   - 基于历史行为排斥恶意客户端

6. 机器反学习(Machine Unlearning)
   - 从全局模型中移除中毒影响
```

**关键创新**:
- ✓ **针对性强**: 专门针对TLFA设计, 不是通用方法
- ✓ **神经元级粒度**: 比梯度级更细致
- ✓ **模型修复**: 不仅检测, 还能恢复

**实验结果**:
- 性能接近无攻击场景
- 比8个基线方法高 **9.49%** (关键指标1) 和 **4.47%** (关键指标2)
- 支持多种恶意客户端比例、数据异构、多任务、动态攻击

**对你的启发**:
- 考虑针对特定攻击类型的专门检测器
- 神经元级分析可能比BN特征更敏感
- 模型修复是后处理的重要步骤

### 2.2 FLARE (2025-11) - 多维声誉评分

**核心思想**: 不是简单的二分类(恶意/良性), 而是**连续的声誉评分**

**方法**:
```
1. 多维声誉评分
   维度1: 性能一致性 (Performance Consistency)
          - 模型精度的稳定性
          - 轮次间的方差

   维度2: 统计异常指标 (Statistical Anomaly)
          - 梯度范数异常
          - 参数分布偏差

   维度3: 时间行为 (Temporal Behavior)
          - 历史评分的EMA
          - 长期可信度

2. 自适应阈值
   - 根据模型收敛状态调整
   - 根据攻击强度动态变化

3. 软排斥(Soft Exclusion)
   - 不是硬删除客户端
   - 而是降低权重: w_i = reputation_i * alpha

4. 差分隐私保护
   - 在评分中加入拉普拉斯噪声
   - 保护客户端隐私
```

**攻击类型测试**:
- 标签翻转
- 梯度缩放(Gradient Scaling)
- 自适应攻击(Adaptive Attacks)
- ALIE (A Little Is Enough)
- **Statistical Mimicry** (新型攻击!)
  - 混合诚实梯度 + 合成扰动
  - 持续漂移以躲避传统过滤器

**实验设置**:
- 100个客户端
- 数据集: MNIST, CIFAR-10, SVHN
- 对比: 最先进的Byzantine-鲁棒方法

**结果**:
- 鲁棒性提升 **16%**
- 收敛速度: 非攻击基线的 **30%** 内
- 计算开销小

**对你的启发**:
- ✓ 多维评分比单一指标更好
- ✓ 软权重比硬排斥更平滑
- ✓ 需要应对自适应攻击(攻击者知道你的防御)
- ✓ 差分隐私是必要的

### 2.3 GShield (2025-12) - 梯度分布聚类

**核心思想**: 学习良性梯度的分布, 然后检测偏离的更新

**方法**:
```
1. 初始化阶段(前N轮)
   - 假设恶意客户端比例 < 50%
   - 收集所有客户端梯度

2. 梯度分布建模
   - 对梯度进行聚类(K-means或GMM)
   - 识别主要聚类(假设为良性)

3. 高斯建模
   - 对主聚类拟合多元高斯分布
   - 学习均值μ和协方差Σ

4. 选择性聚合
   - 计算每个客户端梯度到高斯分布的Mahalanobis距离
   - 距离 < 阈值 → 聚合
   - 距离 > 阈值 → 排斥
```

**攻击类型**:
- 数据投毒(Data Poisoning)
- 模型投毒(Model Poisoning)
- 非IID条件下的攻击

**实验设置**:
- 表格数据 + 图像数据
- 非IID数据分布
- 对比多个防御方法

**结果**:
- 目标类精度提升 **43-65%**
- 整体精度保持高水平

**对你的启发**:
- ✓ 高斯建模比SVDD更简单
- ✓ Mahalanobis距离考虑特征相关性
- ✓ 需要处理非IID数据(你的代码假设IID)

### 2.4 FL-LTD (2026-01) - 损失趋势偏差

**创新点**: 不看梯度, 看**损失函数的变化趋势**

**方法**:
```
1. 监控每个客户端的损失轨迹
   L_i(t) = 客户端i在第t轮的训练损失

2. 计算损失趋势
   trend_i = L_i(t) - L_i(t-1)

3. 异常检测
   - 良性客户端: 损失单调递减
   - 恶意客户端: 损失波动/上升

4. 阈值判断
   - 如果 trend_i > threshold → 标记为恶意
```

**攻击类型**:
- 损失操纵攻击(Loss Manipulation Attacks)
- 非IID MNIST

**优势**:
- ✓ 无需梯度检查(计算快)
- ✓ 对某些攻击更敏感
- ✓ 轻量级

**对你的启发**:
- 可以作为辅助指标
- 与BN特征结合可能更强

---

### 2.5 TinyGuard (2026-02) - 统计指纹

**核心思想**: 提取**紧凑的统计指纹**, 无需全维分析

**方法**:
```
1. 统计指纹提取
   - 梯度范数: ||g_i||_2
   - 梯度方差: Var(g_i)
   - 参数范数: ||θ_i||_2
   - 参数方差: Var(θ_i)
   - 其他统计量...

2. 指纹向量
   f_i = [||g_i||_2, Var(g_i), ||θ_i||_2, Var(θ_i), ...]

3. 异常检测
   - 使用LOF或Isolation Forest
   - 或简单的MAD阈值
```

**优势**:
- ✓ 轻量级(只需几个统计量)
- ✓ 快速计算
- ✓ 易于部署

**对你的启发**:
- BN特征可能过于复杂
- 简单的统计指纹可能足够
- 考虑特征选择

---

### 2.6 SpectralKrum (2025-12) - 谱几何混合

**方法**:
```
1. 谱子空间估计
   - 对所有客户端梯度进行SVD
   - 提取主子空间

2. 几何邻域选择
   - 在子空间中找到最近邻
   - 类似Krum但在降维空间

3. 聚合
   - 选择邻域内的梯度
   - 加权平均
```

**优势**:
- ✓ 异构设置下鲁棒
- ✓ 结合谱方法和几何方法

### 4.1 从2024到2026的演变

**2025年**:

**2026年**:

- 区块链验证(PenTiDef)
- 差分隐私集成

### 4.2 新型攻击

**Statistical Mimicry** (FLARE中提出):
```
攻击策略:
1. 混合诚实梯度 + 合成扰动
2. 持续漂移以躲避传统过滤器
3. 目标: 绕过所有现有防御

防御策略:
- 多维评分(不仅看梯度)
- 时间序列分析(检测漂移)
- 自适应阈值
```

**对你的启发**:
- 你的BN特征可能对这种攻击更鲁棒
- 因为BN特征反映的是模型行为, 不是梯度本身

### 4.3 共同趋势

1. **从硬排斥到软权重**
   - 旧: 检测到恶意 → 删除
   - 新: 检测到恶意 → 降低权重

2. **多维评分**
   - 旧: 单一指标(梯度范数)
   - 新: 多个维度(性能+统计+时间)

3. **隐私保护**
   - 旧: 无隐私考虑
   - 新: 差分隐私集成

4. **自适应防御**
   - 旧: 固定阈值
   - 新: 动态调整

5. **非IID数据**
   - 旧: 假设IID
   - 新: 显式处理非IID

```python
# Statistical Mimicry 攻击
def statistical_mimicry_attack(global_model, local_model, alpha=0.5):
    """
    混合诚实梯度 + 合成扰动
    """
    honest_grad = local_model - global_model
    noise = torch.randn_like(honest_grad) * 0.1
    drift = 0.01 * torch.randn_like(honest_grad)  # 持续漂移

    attack_grad = alpha * honest_grad + (1-alpha) * noise + drift
    return global_model + attack_grad
```
